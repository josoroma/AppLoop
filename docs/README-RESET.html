AppLoop · Ops runbook

How to run, reset, and seed from scratch

This assumes dependencies and env are already set up (Hermes installed, root .env / .hermes/.env present). It documents the Makefile path to boot the builder, wipe local AppLoop state, reinstall every template’s dependencies, migrate SQLite, and seed one demo project per built-in template.

Builder make dev → :3001 Gateway make hermes-gateway → :8642 Full clean start make seed Source of truth Makefile
Daily run
gateway + dev

Two terminals: Hermes API server, then AppLoop Next.js builder.

Soft reseed
make seed

Wipe .apploop + template node_modules, migrate DB, install templates, seed demos. Keeps root node_modules.

Hard teardown
make reset

Also deletes root node_modules and lockfile. Follow with make install && make apploop-seed.

01 · Assumptions

Prerequisites (already set up)

This runbook starts after first-time setup. You should already have:

  • Node.js + npm on PATH
  • AppLoop repo checked out; typically root deps installed via make install once
  • hermes CLI available for gateway mode
  • Local secrets in .env and/or .hermes/.env (gateway key, model/provider as needed)
  • Working directory = repo root (where Makefile lives)
Do not casually source production secrets into untrusted shells. When validation prompts ask to source .env, answer N unless the task needs local secrets. Makefile Hermes targets source .hermes/.env then .env themselves.

02 · Run

Daily: run the stack

Chat-driven edits need the Hermes gateway up. The builder alone is enough for browsing projects, but inspect → send path will error without gateway connectivity.

make hermes-gateway

Terminal A — AI gateway

Starts Hermes with HERMES_HOME=.hermes at http://127.0.0.1:8642 (override via HERMES_GATEWAY_PORT / API_SERVER_PORT). Reuses API_SERVER_KEY from env or default local key. Stops an existing Hermes listener on that port when safe.

make hermes-gateway-curl-test

Optional smoke test

Confirms the gateway is reachable and can accept a minimal POST /v1/runs.

make dev

Terminal B — AppLoop builder

Starts Next.js at http://localhost:3001 (PORT override supported). If something Next/npm/node-owned already holds the port, Makefile stops it first. 

# Typical two-terminal session
cd /path/to/AppLoop

# Terminal A
make hermes-gateway

# Terminal B
make dev
# → open http://localhost:3001/projects

Also useful while developing

make check

lint + typecheck

Fast gate before bigger UI/domain changes.

make build

production build

Builder production bundle only (not all templates).

make hermes-dashboard

Hermes dashboard :9120

Optional operator UI for Hermes itself.

03 · Makefile orthography

Target ladder (what each make target does)

Read this before picking a destructiveness level. Targets compose.

Target Does Does not
make clean Removes builder .next, out, dist, build, tsconfig.tsbuildinfo Leave projects/DB/templates alone
make apploop-reset Deletes .apploop/projects, .apploop/runtime-logs, SQLite files under .apploop/; strips every templates/*/{node_modules,package-lock.json,pnpm-lock.yaml} Remove root deps; delete template source code; wipe .hermes/
make apploop-template-seed npm install in each templates/*/ that has package.json Touch SQLite or create projects
make apploop-seed-projects Depends on template-seed → npm run db:migrate → npx tsx scripts/seed-projects.mts Wipe existing demos with same names (skips duplicates)
make apploop-seed Alias end of seed-projects with success banner Reset first
make seed Recommended clean start: apploop-reset then apploop-seed Remove root node_modules
make reset clean + apploop-reset + delete root node_modules + root lockfile Reinstall anything (you must make install after)

04 · Current catalog

Seed what we have now

Two different “template sets” matter. Do not conflate them.

A. Disk templates (templates/*)

make apploop-template-seed installs deps for every directory under templates/ that contains a package.json. Currently that includes:

default admin-luma ai-engineer-cv deep-research-paper luminous-rings solar-system algovivo-creature immersive-full-screen lumacv vestaboard

If you add another template folder with a package.json, the loop picks it up automatically.

B. Demo projects created by seed script

scripts/seed-projects.mts creates one project per entry in BUILT_IN_PROJECT_TEMPLATES (lib/projects/templates.ts), not every disk folder. Registry today:

idProject name seededTheme
defaultDefault Next.jsluma-blue-violet
admin-lumaLuma Adminluma-admin-amber
ai-engineer-cvAI Engineer CVluma-cv-indigo
deep-research-paperDeep Research Paperluma-amber-slate
luminous-ringsLuminous Ringsluma-violet-cyan
solar-systemSolar System Explorerluma-indigo-emerald
algovivo-creatureAlgovivo Soft Creatureluma-orange-stone
Important catalog gap: folders like templates/vestaboard and templates/lumacv get npm install during template-seed, but they are not in BUILT_IN_PROJECT_TEMPLATES, so make seed will not auto-create demo projects for them. They still appear as create-project template options only after they are registered in lib/projects/templates.ts (or as ready custom templates in SQLite). Until then, open them via Templates tooling / create flows that know those paths, or add them to the built-in registry and re-run seed.

What each seeded demo project materializes

For each built-in template not already present by name:
  ProjectService.createProject({ name: tpl.name, themeId: tpl.defaultThemeId })
    → SQLite: projects, conversations, runtimes, project_themes, project_settings
  createProjectWorkspace(PROJECTS_ROOT, workspacePath, { template, theme })
    → FS copy templates/ → .apploop/projects/
    → apply theme CSS
    → git init in workspace

Env defaults (overridable):
  PROJECTS_ROOT=file path → .apploop/projects
  DATABASE_URL=file:.apploop/builder.sqlite
  PREVIEW_PORT_START=3100
  PREVIEW_PORT_END=3199

05 · Recipes

From-scratch recipes

Recipe 1 — Recommended: clean AppLoop state, keep root deps

Use this when projects/DB are dirty but the monorepo install is fine.

  1. Stop builder/gateway/preview processes if they hold ports (Ctrl-C; or let make dev reclaim 3001).
  2. Run the full soft reseed:
make seed
# expands to:
#   make apploop-reset
#     rm -rf .apploop/projects .apploop/runtime-logs
#     rm -f  .apploop/*.sqlite .apploop/*.db .apploop/*.sqlite.backup
#     rm -rf templates/*/node_modules templates/*/package-lock.json …
#   make apploop-seed
#     make apploop-template-seed   # npm install in every templates/*
#     npm run db:migrate
#     npx tsx scripts/seed-projects.mts
  1. Start services:
make hermes-gateway   # terminal A
make dev              # terminal B
# open http://localhost:3001/projects

Recipe 2 — Nuclear: dependencies and all local AppLoop state

Destructive. Deletes root node_modules and the root lockfile. Expect a longer reinstall. 
make reset
# = clean + apploop-reset + rm -rf node_modules package-lock.json

make install
make apploop-seed
make hermes-gateway
make dev

Recipe 3 — Incremental: only missing demos

If DB already has migrations and some demos exist, seed is idempotent by project name.

make apploop-template-seed   # refresh template installs if needed
make apploop-seed-projects   # skips templates whose demo name already exists
make dev

Recipe 4 — Templates only (no project reseed)

make apploop-template-seed
# useful after adding a new templates//package.json

06 · Safety

What gets deleted vs preserved

Deleted by make seed / apploop-reset

  • All generated workspaces under .apploop/projects/
  • Runtime logs under .apploop/runtime-logs/
  • SQLite DBs at .apploop/*.sqlite, *.db, backups
  • Per-template node_modules and lockfiles
  • Any unsaved chat history that lived only in those DB files

Preserved

  • Repo source: app/, lib/, components/, templates/*/ source
  • Custom template source trees under templates// (code remains; their node_modules are wiped)
  • .hermes/ config, skills, agents (gateway home)
  • Root node_modules on make seed (removed only by make reset)
  • Root .env / secrets
Custom template DB rows live in SQLite under .apploop/. After apploop-reset/seed, those rows are gone. Disk folders under templates/ may still exist as orphan trees until you delete them manually or recreate rows via Create Template / register again.

07 · Verify

Verification checklist

# 1) Template installs present
for d in templates/*/; do
  [ -d "${d}node_modules" ] && echo "OK  $d" || echo "MISS $d"
done

# 2) DB file exists after migrate/seed
ls -la .apploop/*.sqlite 2>/dev/null || ls -la .apploop

# 3) Demo workspaces exist
ls .apploop/projects

# 4) Builder responds
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:3001/projects

# 5) Gateway responds (if running)
make hermes-gateway-curl-test

In the UI after make seed + make dev, /projects should list the six seeded demo projects named exactly like the built-in template display names (Default Next.js, Luma Admin, AI Engineer CV, Deep Research Paper, Luminous Rings, Solar System Explorer).

08 · Pitfalls

Pitfalls & notes

PitfallDetail
Seed skipped “already exists” seed-projects.mts skips when a project with the same name already exists. Use make seed (which resets DB/projects) for a true clean slate.
make dev double port flag Makefile already passes --port $(PORT). Keep the port out of package.json dev scripts to avoid next dev --port 3001 --port 3001.
Gateway up but builder chats fail Confirm AppLoop env points at the same base URL/key the gateway uses (HERMES_BASE_URL / API server key alignment).
Template seed is slow Installing deps for every template can take minutes. That is expected on first seed after reset.
Orphan template folders Custom/vestaboard/lumacv source remains on disk after DB wipe. Preview projects from seed only cover built-in registry entries.
make reset without reinstall Builder will not start until make install (and usually make apploop-seed).
Preview ports occupied After a hard kill, old next dev processes on 3100–3199 may linger. Free them before seeding many projects if allocation fails.
