## Vendored runtime

`public/vendor/algovivo/` (jsDelivr pin example `@d457bbc/build/...`):

| Asset | Role |
|-------|------|
| `algovivo.min.js` / `.wasm` | ESM + WASM engine |
| `lib/creature-mesh.ts` | Bundled mesh snapshot (`pos/triangles/muscles/l0/rsi/depth`) — prefer over boot-time fetch |
| Optional `public/data/.../mesh.json` | Human-readable dump / older assets |

Client load:

```ts
const href = new URL('/vendor/algovivo/algovivo.min.js', window.location.origin).href
const mod = await import(/* webpackIgnore: true */ href)
// WASM: instantiateStreaming first; fallback arrayBuffer + instantiate

// CRITICAL: include rsi or the body collapses to sticks
system.set({
  pos: mesh.pos.map((p) => [...p]),
  triangles: mesh.triangles.map((t) => [...t]),
  muscles: mesh.muscles.map((m) => [...m]),
  l0: [...mesh.l0],
  rsi: mesh.rsi.map((cell) => cell.map((row) => [...row])),
})

// Normal speed: 30Hz timestep (default). Slower = more elastic/thrashy.
system.h = 0.033

const viewport = new SystemViewport({
  system,
  borderColor: '#0a0a0a', // black mesh edges (user rejected blue neon)
  fillColor: '#f5c400',
  // Universe look: transparent canvas, starfield painted by CSS behind it.
  gridColor: 'rgba(0, 0, 0, 0)',
  backgroundColor: 'rgba(0, 0, 0, 0)',
  activeMuscleColor: '#ff2a3a',
  inactiveMuscleColor: '#8a1520',
  vertexDepths: [...mesh.depth],
  draggable: true,
})

viewport.tracker.visibleWorldWidth = 6.4
viewport.tracker.targetCenterY = 1.05
// Floor reuses borderColor — mute or black edges paint a black horizon.
if (viewport.floor?.mesh) {
  viewport.floor.mesh.lines = []
  if (viewport.floor.mesh.lineShader) viewport.floor.mesh.lineShader.renderLine = () => {}
}
```

**Black-edges legend on dark HUD:** `.legend-joint` `#0a0a0a` needs a white ring (`.legend-joint::before { box-shadow: 0 0 0 1.5px rgba(255,255,255,0.92), 0 0 8px rgba(255,255,255,0.28); }`) — user: *“black edges text should have white border”*.

Muscle API: `a[i] === 1` relaxed; `< 1` contracts toward fraction of rest length `l0`.

## CRITICAL pitfall — stick-figure collapse

**Symptom:** sparse edge graph; no filled soft body.

**Cause:** `system.set` without **`rsi`**. Neo-Hookean triangles need rest-shape tensors.

**Fix:** When hand-authoring, **recompute rsi** per triangle:

```python
Dm = np.column_stack([pb - pa, pc - pa])
rsi = np.linalg.inv(Dm)  # skip near-zero det
```

Official quadruped `mesh.json` remains the gold reference for a trained walk (pos, triangles, muscles, l0, rsi, depth).

## CRITICAL pitfall — anxious / double-speed motion

**Symptom:** thrashy motion; user says "slow down" / "anxious".

**Cause:** `system.step()` on every `requestAnimationFrame` (~60 Hz) while nominal engine loop is ~30 Hz (`h ≈ 0.033`).

**Fix — fixed-step accumulator (hardened for cold start):**

```ts
const SIM_HZ = 30
const SIM_DT_MS = 1000 / SIM_HZ
const MAX_SIM_STEPS_PER_FRAME = 1   // 2 + lag dump = "starts erratically fast"
const MAX_SIM_LAG_MS = SIM_DT_MS * 1.5
const SETTLE_STEPS = 8              // rest a=1 before first policy.step()
// settle once after system.set + policy load, then zero the RAF clock
// if (now - lastSimAt > MAX_SIM_LAG_MS) lastSimAt = now - SIM_DT_MS
// while (now - lastSimAt >= SIM_DT_MS && steps < MAX_SIM_STEPS_PER_FRAME) { gait(); system.step() }
```

User: *“sometimes creature starts erratically fast”* → lag catch-up thrash, not permanent walk speed.

Controller notes:

- **Scripted gaits don't produce real locomotion — measured.** 40+ headless sweeps of scripted `system.a.set` walks (lateral/diagonal/trot phases, retractor levers, shear-diagonal leg muscles) all yielded ≤0.007 u/s forward, some drifting backward. Official quadruped + pretrained MLP policy: **1.71 u/s**. If the user asks the creature to "move forward", switch to the policy — don't keep tuning sine gaits.
- **Neural-walker cat recipe (current template):** run `scripts/generate-cat-mesh.py` — it appends ears + a compact tail AFTER the quadruped's 62 verts / 38 muscles and recomputes `l0`/`rsi` for appended geometry only (verifies zero rsi drift on originals). The scene loads `/data/quadruped/policy.json` and constructs `new nn.MLPPolicy({ system, active: true, numVertices: POLICY_VERTEX_COUNT, numMuscles: POLICY_MUSCLE_COUNT })`. **`active: true` is mandatory — MLPPolicy defaults inactive and silently outputs a=1.**
- Per fixed step: `policy.step()` → read `system.a.toArray()` → `applyTailSway()` rewrites only appended tail slots (anchors pinned at 1) → `system.a.set(...)` → `system.step()`.
- **Appendage inertia tax:** tall raised tail dropped the trained walk 1.71→0.26 u/s; compact low 3-vert tail keeps ~0.3–0.96 u/s depending on mount; ear height barely matters. Sweep appendage size headlessly before shipping.
- **Tail mount (user: “more in the ass or back”):** do **not** leave the append on mid-spine rear-top `(0,1)`. Mount on rear rump edge **`(3,4)`** so the tail sits on the ass/backside. `scripts/generate-cat-mesh.py` must match; regenerate after any move.
- Custom meshes without the policy → scripted `system.a.set` is display-only motion (stepping in place, tail sway), not locomotion.
- Legacy scripted-gait tuning (for in-place motion): diagonal leg phases speed ~1.3, activations clamped ≥0.74, body stabilizers 0.96–0.98, tail root anchors exactly 1.

## Universe starfield stage (current background)

User rejected the engine grid + brown page tones — the stage must read as deep space with neon accents. Recipe:

1. **Make the sim canvas transparent** via `SystemViewport` options: `gridColor: 'rgba(0,0,0,0)'`, `backgroundColor: 'rgba(0,0,0,0)'` (single `backgroundColor` overrides the center/outer radial pair; the renderer `clearRect`s each frame so alpha persists). The grid/floor still exist but are invisible; the blue floor line comes from `borderColor` on the floor renderer.
2. **Paint space in CSS** on the host (`.algovivo-viewport`): base `radial-gradient(circle at 50% 40%, #0b1026 0%, #070a18 55%, #030510 100%)` + 2–3 oklch nebula ellipses (hues 220/300/330, ~20–30% alpha).
3. **Two star layers** as `::before`/`::after` with `inset: -50%`, each a stack of tiny `radial-gradient(<size> at x% y%, color, transparent)` dots tiled by `background-size` (~34% far / ~48% near). Far layer: 1.4–1.5px faint white/blue. Near layer: 2.1–2.8px neon-tinted (cyan/pink/lilac/amber) with a twinkle opacity animation. Opposite-direction slow `translate3d` drift (140s / 90s reverse); guard with `prefers-reduced-motion`.
4. Canvas gets `position: relative; z-index: 1` so drag events still hit it above the pseudo-elements (`pointer-events: none` on both).
5. **First pass looked too faint in screenshots** — bump star sizes ≥1.4px, alpha ≥0.65, and shrink `background-size` (denser tiling) until stars are clearly visible in a `browser_vision` capture.
6. Kill every warm/brown token while at it: body bg, `.algovivo-shell` gradients, `--background`/`--card` all live in oklch hues 220–320.

## CRITICAL pitfall — appendage (tail/ear) detaches and falls

**Symptom:** tail (or another appendage) visually separates from the body and drops as a loose circle/part; user reports "a circle falls down from the cat".

**Cause:** appendage is a chain hanging from a single attachment vertex, AND its muscles (including the root segment) contract in the gait. Contraction pulls the single anchor point away from the body; with nothing else binding it, the part tears off and falls.

**Fix (both required):**

1. **Structural anchoring** — put the appendage's first vertex *inside* the parent body region (overlapping the rump/torso area), and triangulate it to **multiple** body points (e.g. tail base bound to spine[0], belly[0], AND spine[1]). One attachment triangle is not enough.
2. **Gait anchoring** — add root-anchor muscles between the body and the appendage base, and hold those indices at activation **exactly 1** in the controller. Only free chain segments may sway, and only near rest length (~0.88–1.0, mild amplitude).

```ts
// tail chain: first 2 muscles are root anchors — rigid
creatureMeta.trunk_muscles.forEach((idx, i) => {
  if (i < 2) { activations[idx] = 1; return }   // root: never contract
  const t = i / (creatureMeta.trunk_muscles.length - 1)
  activations[idx] = clamp(0.93 + 0.06 * wave * (0.3 + t * 0.7), 0.88, 1)
})
```

## Mesh authoring loop

1. Silhouette primitives (convex body fan around a core vertex, thin leg strips, ears/tail/head geometry).
2. Fan non-degenerate triangles for volume.
3. Muscles only on actuators; keep group meta aligned with index order.
4. Emit `l0` + `rsi` + optional `depth`.
5. Write `lib/creature-mesh.ts` (+ optional JSON dump).
6. Typecheck + rsync seeded workspace.

Neural-walker cat (~67v/100t/41m): official quadruped body + ears on head-top edges `(28,23)` / `(27,8)` + compact raised tail on **rump `(3,4)`** (not spine `(0,1)`). Hand-sparsed cat mesh is obsolete for locomotion.

## Pitfalls

- Dynamic import **without** `webpackIgnore` breaks `/vendor` resolution.
- Missing vendored WASM in workspace after seed → boot fails.
- `system.a.set` length must equal muscle count.
- Re-adding face overlays after user said eyes/mouth are not needed — don't.
- Leaving HUD as absolute floating card after user asked for left-column info.
- Using default `visibleWorldWidth` (~3.8) when user wants a smaller subject.
- rAF-tied physics after "anxious" feedback.
- Assuming pretrained policy works on any hand mesh.
- **High vertex count / dense meshes = "noise".** Keep silhouettes sparse (~40 verts), one muscle chain per leg joint, body stabilizers near 0.96–0.98.
- Disabling drag (`draggable: false`) — user explicitly asked to keep it.
- Slowing sim below 16Hz or `h` below 0.022 — motion becomes elastic/thrashy. Normal = 30Hz + `h=0.033`.
- Leaving max-2 catch-up after a heavy first paint — *“starts erratically fast”*; use max-1 + lag clamp + rest settle + clock reset on walk entry.
- Transparent grid but still drawing floor while edges are black — thick black horizon; mute `viewport.floor` after boot.
- Black legend swatch with no white outline — invisible on the dark HUD.
- Status cell using the same `.algovivo-stat-value` size as numbers — `walking` overflows; shrink via `.creature-stat-status-value`.
- Tail glued to mid-spine rear-top `(0,1)` after user asked for ass/rump — reattach to `(3,4)`.
